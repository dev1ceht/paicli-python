# Inject context variants into the production Agent path

Live context comparisons will use the same PaiCLI runtime and production `QueryEngine`/Agent path, injecting either a full-history context manager or the current optimized production `ContextManager` at Agent construction while leaving the production default unchanged. The full-history manager performs no reduction but retains equivalent telemetry and the same request-budget enforcement; the optimized manager may only receive the uniform versioned profile, not per-task tuning, and every model-assisted summary call belongs to its provider input-token cost. This isolates context policy from unrelated source-version differences and avoids a benchmark-only Agent loop, at the cost of adding a narrow context-manager construction seam to production orchestration.
