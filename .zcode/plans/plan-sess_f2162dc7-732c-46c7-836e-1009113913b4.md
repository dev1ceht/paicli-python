## 渲染系统升级计划

按照 Java 版 InlineRenderer 的样式，对 Python 版渲染系统进行以下增强：

### 1. `src/paicli/render/rich_renderer.py` - 核心重写
- **工具标签系统**: 中文 emoji 标签 (📖 读取、✏️ 写入、⚡ 执行命令、🔍 搜索代码等)
- **成本估算**: `estimate_cost()` 根据 provider 计算费用 (¥X.XXXX)
- **耗时追踪**: `time.monotonic()` 追踪每次 run 的耗时
- **Token 格式化**: `format_tokens()` 输出 51.0k / 1.2M 格式
- **耗时格式化**: `format_elapsed()` 输出 250ms / 1.5s 格式
- **Phase 追踪**: idle/running/plan 三种状态
- **回答标记**: ▪ 符号在最终输出前显示
- **Diff 渲染**: LCS 算法 + 上下文裁剪 + 颜色输出
- **Banner 增强**: Workspace 面板显示 Model/HITL/Tools/MCP/Skills/CWD
- **NO_COLOR 支持**: 检测 NO_COLOR 环境变量
- **Cached tokens 追踪**: 记录缓存 token 数
- **toolbar_status()**: 新增 elapsed/cost/phase/cached_tokens 字段

### 2. `src/paicli/entrypoints/repl.py` - 工具栏增强
- 底部工具栏新增 in/out/cache token 计数
- 新增成本显示和耗时显示
- 新增 phase 状态显示
- 新增 HITL 状态指示
- 调用 `renderer.set_provider()` 传递 provider

### 3. `src/paicli/render/__init__.py` - 更新导出
- 导出 `format_tokens`, `format_elapsed`, `estimate_cost`

### 4. `tests/test_render.py` - 测试扩展
- 工具标签测试
- 成本估算测试
- Token 格式化测试
- Diff 渲染测试
- 新增 toolbar_status 字段测试
- Banner workspace 面板测试
