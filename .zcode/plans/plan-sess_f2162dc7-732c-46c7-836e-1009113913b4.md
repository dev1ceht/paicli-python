## 完整上下文治理系统实现计划

四阶段上下文治理 + 工具结果裁剪，融合为一个完整系统。

### 阶段一：动态预算计算 (`budget.py`)

```python
def calculate_budget(context_window: int, *, utilization: float = 0.50,
                     output_reserve: int = 4096) -> Budget:
    """
    1. 可用 token = context_window - output_reserve
    2. prompt_budget = 可用 token * utilization（默认 0.50）
    3. char_budget = prompt_budget * 4（1 token ≈ 4 chars）
    4. 上下界保护：min(800_000, max(60_000, char_budget))
    """
```

### 阶段二：分层组装 (`assembler.py`)

按固定顺序组装，分配预算：

| Section | 预算占比 | 裁剪优先级 | 说明 |
|---|---|---|---|
| prefix | 15% | 最低 | 系统指令、工作区上下文，利于 cache 命中 |
| memory | 10% | 低 | 工作记忆、todo、checkpoint |
| skills | 10% | 中 | 技能描述，固定文本可重载 |
| relevant_memory | 10% | 最高 | 向量检索命中，下一轮可重取 |
| history | 45% | 中高 | 对话历史 + 工具结果（含工具结果裁剪） |
| current_request | 10% | 不裁剪 | 当前用户输入，永远完整保留 |

**工具结果裁剪在 history 层内部执行**：

```
history 层组装流程：
1. 旧工具结果压缩 → 保留最近 5 条完整，更早的替换为占位符
2. 大工具结果落盘 → 超过 200KB 的保存到磁盘，保留预览
3. 常规截断 → 按 section 预算裁剪
```

### 阶段三：压力感知裁剪 (`pressure.py`)

```python
pressure_ratio = max(rendered_tokens, raw_tokens) / budget_tokens

# 四个等级
< 60%    → tier0: 不处理
60-80%   → tier1: relevant_memory 缩到 70%
80-95%   → tier2: tier1 + skills 缩到 50%
>= 95%   → tier3: 触发 compaction

# 溢出兜底（compaction 后仍超预算）
裁剪顺序: relevant_memory → skills → history → memory → prefix
```

### 阶段四：结构化压缩 (`compaction.py`)

**触发条件**:
- tier3 且 delta items >= 6
- 或 prompt 已超预算

**两种摘要模式**:

1. **LLM 摘要**（常规）：
   - 最近 2 个 turn 保护不压缩
   - delta items 送入 LLM 生成结构化摘要
   - 输出格式：Goal / Constraints / Files Read/Modified / Key Decisions / Blockers / Next Steps

2. **确定性摘要**（兜底）：
   - 不调用模型，纯规则提取
   - 正则匹配 Goal / Constraints / Files / Decisions / Errors
   - 硬限 2000 字符

**增量合并**: 旧摘要 + 新 delta → 合并摘要

```python
def compress_old_tool_results(
    messages: list[Message],
    *,
    keep_recent: int = 5,

    """
    1. 找出所有 role="tool" 的消息
    2. 保留最近 keep_recent 条完整内容
    3. 更早的替换为占位符：
       "[工具结果已压缩，tool_call_id=xxx，需要时重新执行]"
    """
```

**策略二：大工具结果落盘**

```python
def offload_large_tool_results(
    messages: list[Message],
    *,
    max_total_bytes: int = 200 * 1024,  # 200KB
    preview_chars: int = 200,
    storage_dir: str = "~/.paicli/tool_results",
) -> list[Message]:
    """
    1. 统计所有工具结果总大小
    2. 超过预算时按大小排序
    3. 最大的保存到磁盘: storage_dir/{session_id}/{tool_call_id}.txt
    4. 上下文保留占位符:
       "[大工具结果已落盘，路径={path}，预览={前200字符}...]"
    """
```

### Token 估算 (`token_estimator.py`)

```python
def estimate_tokens(text: str) -> int:
    """启发式估算"""
    chinese = sum(1 for c in text if '\u4e00' <= c <= '\u9fff')
    code = sum(1 for c in text if c in '{}[]();=')  # 代码特征字符
    other = len(text) - chinese - code
    return ceil(chinese / 1.8 + code / 3.2 + other / 4.0)

def calibrate(estimated: int, actual: int) -> float:
    """用实际 usage 校准比例因子"""
    if estimated == 0:
        return 1.0
    return actual / estimated
```

```python
# query.py - 每轮 LLM 调用前
async def query(...):
    while turn < max_turns:
        # 构建 prompt（含工具结果裁剪）
        prompt = context_manager.build_prompt(
            history=messages,
            user_message=user_message,
            actual_usage=last_actual_usage,  # 校准
        )
        # LLM 调用
        async for event in llm_client.chat(messages, tools, system_prompt=prompt):
            ...
        # 记录 actual_usage
        context_manager.record_usage(usage_input, usage_output)
```
