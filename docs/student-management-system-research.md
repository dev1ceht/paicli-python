# 学生管理系统需求调研报告

> 调研时间：2026-07-09
> 调研范围：功能需求、用户角色、技术架构、数据库设计

---

## 1. 项目背景与目标

学生管理系统（Student Management System, SMS）是一套面向学校/教育机构的综合信息管理平台，旨在替代传统人工管理方式，实现学生信息、课程、成绩、考勤、缴费等教务管理工作的数字化、自动化。

**核心目标：**
- 提高教务管理效率，降低人力成本
- 实现数据的集中存储与实时共享
- 加强学校、教师、学生、家长四方的沟通协作
- 提供数据分析和决策支持

---

## 2. 用户角色分析

| 角色 | 描述 | 核心权限 |
|------|------|----------|
| **系统管理员** | 学校 IT/教务管理人员 | 系统配置、角色权限分配、全校数据查看、日志审计 |
| **教务管理员** | 教务处工作人员 | 学生学籍管理、课程编排、考试安排、成绩总览、教师分配 |
| **教师** | 授课教师/班主任 | 成绩录入、考勤记录、课程管理、学生评语、查看课程表 |
| **学生** | 在校学生 | 查看个人信息、选课/退课、查成绩、查课表、请假申请 |
| **家长** | 学生监护人 | 查看子女成绩/考勤/课表、接收通知、与教师沟通 |
| **财务人员** | 学校财务部门 | 学费管理、缴费记录、发票开具、欠费催缴 |

## 3. 功能需求

### 3.1 用户认证与权限管理

| 功能 | 描述 |
|------|------|
| 登录/登出 | 用户名+密码登录，支持验证码 |
| 密码重置 | 邮箱/手机验证找回密码 |
| 角色权限控制 | RBAC（基于角色的访问控制），精确到按钮级别 |
| 多角色切换 | 一个账号可绑定多个角色 |
| 操作日志 | 记录所有用户的关键操作，支持审计 |

### 3.2 学生信息管理

| 功能 | 描述 |
|------|------|
| 学生档案 | 姓名、学号、性别、出生日期、民族、政治面貌、籍贯、照片等 |
| 联系信息 | 手机号、邮箱、家庭地址、紧急联系人 |
| 学籍管理 | 入学、休学、复学、转专业、退学、毕业等学籍变动记录 |
| 批量导入/导出 | 支持 Excel/CSV 批量导入导出学生信息 |
| 文档管理 | 上传/下载学生相关文档（如成绩单、证明文件） |
| 高级检索 | 按条件（班级、年级、专业、状态等）快速筛选 |

### 3.3 课程管理

| 功能 | 描述 |
|------|------|
| 课程信息 | 课程名称、编码、学分、学时、上课地点、授课教师 |
| 课程分类 | 按院系、专业、年级分类管理 |
| 课程编排 | 排课、教室分配、时间冲突检测 |
| 培养方案 | 专业培养计划、必修/选修课程体系 |
| 课程公告 | 发布课程相关通知 |

### 3.4 选课管理

| 功能 | 描述 |
|------|------|
| 选课设置 | 设置选课时间窗口、容量上限、选课条件 |
| 学生选课/退课 | 在线选课、退课、查看已选课程列表 |
| 冲突检测 | 自动检测时间、考试冲突 |
| 选课统计 | 各课程选课人数、余量实时统计 |
| 补考管理 | 补考安排、补考成绩记录 |

### 3.5 成绩管理

| 功能 | 描述 |
|------|------|
| 成绩录入 | 教师录入平时成绩、期中/期末考试成绩、实验成绩 |
| 批量录入 | 支持 Excel 批量导入成绩 |
| 成绩计算 | 按权重自动计算总评成绩（支持 GPA、百分制、等级制） |
| 成绩查询 | 学生查个人成绩、教师查班级成绩、管理员查全校成绩 |
| 成绩分析 | 成绩分布统计、排名、趋势图表 |
| 成绩复查 | 学生申请成绩复查，教师审核处理 |
| 成绩单 | 自动生成可打印/导出的成绩单 |

### 3.6 考勤管理

| 功能 | 描述 |
|------|------|
| 课堂考勤 | 教师通过系统记录出勤/缺勤/迟到/早退 |
| 请假管理 | 学生在线提交请假申请，教师/管理员审批 |
| 考勤统计 | 按学生/班级/课程统计出勤率，生成报表 |
| 自动预警 | 缺勤过多自动通知班主任和家长 |
| 签到方式 | 支持二维码签到、GPS 签到等（可选） |

### 3.7 考试管理

| 功能 | 描述 |
|------|------|
| 考试安排 | 考试时间、地点、监考教师安排 |
| 考场分配 | 自动分配考场和座位号 |
| 成绩录入 | 对接成绩管理模块 |
| 补考安排 | 不及格学生自动进入补考名单 |

### 3.8 通知与沟通

| 功能 | 描述 |
|------|------|
| 系统通知 | 课程变动、考试安排、放假通知等 |
| 消息推送 | 站内信、邮件、短信、App 推送 |
| 家校沟通 | 教师与家长之间的双向消息 |
| 公告板 | 学校/院系/班级级公告发布 |

### 3.9 财务管理

| 功能 | 描述 |
|------|------|
| 学费管理 | 学费标准设置、应缴/实缴/欠费记录 |
| 在线缴费 | 对接支付平台，支持线上缴费 |
| 发票管理 | 电子发票开具/下载 |
| 缴费统计 | 按年级/专业统计缴费率 |

### 3.10 课表管理

| 功能 | 描述 |
|------|------|
| 教师课表 | 查看个人授课时间表 |
| 学生课表 | 查看个人课程安排 |
| 班级课表 | 班级整体课程安排视图 |
| 教室课表 | 教室使用状态查看 |
| 冲突检测 | 自动检测排课冲突 |

### 3.11 系统管理

| 功能 | 描述 |
|------|------|
| 用户管理 | 创建/编辑/禁用用户账号 |
| 角色管理 | 自定义角色和权限分配 |
| 菜单管理 | 动态配置系统菜单 |
| 数据字典 | 维护系统基础数据 |
| 日志审计 | 操作日志、登录日志 |
| 系统配置 | 系统参数、界面风格配置 |

### 3.12 数据统计与报表

| 功能 | 描述 |
|------|------|
| 综合报表 | 学生人数统计、成绩分析、就业率等 |
| 图表展示 | 柱状图、折线图、饼图等可视化 |
| 数据导出 | PDF/Excel 报表导出 |
| 大屏展示 | 学校管理数据大屏（可选） |

### 3.13 移动端

| 功能 | 描述 |
|------|------|
| 移动端适配 | 支持 iOS/Android App 或微信小程序 |
| 消息推送 | 实时接收系统通知 |
| 移动办公 | 移动端成绩审批、请假审批、课表查看 |

## 4. 非功能需求

| 类别 | 需求 |
|------|------|
| **性能** | 并发支持 ≥ 500 用户同时在线；页面响应 < 2s |
| **安全性** | 数据传输加密（HTTPS）、密码加密存储、防 SQL 注入、防 XSS 攻击 |
| **可用性** | 7×24 小时运行，系统可用性 ≥ 99.9% |
| **可扩展性** | 模块化设计，支持功能扩展和第三方集成 |
| **兼容性** | 支持主流浏览器（Chrome、Edge、Firefox、Safari） |
| **可维护性** | 代码规范、文档完善、日志完整 |
| **数据备份** | 自动定期备份，支持数据恢复 |

## 5. 推荐技术架构

### 5.1 前后端分离架构

```
┌─────────────────────────────────────────────────┐
│                   客户端层                        │
│   Web (Vue/React)  │  移动端 App  │  小程序      │
└───────────────────────┬─────────────────────────┘
                        │ HTTP/WebSocket
┌───────────────────────▼─────────────────────────┐
│                 API 网关层                        │
│         身份认证  │  限流  │  路由  │  日志       │
└───────────────────────┬─────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────┐
│               业务服务层                          │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌─────┐  │
│  │学生  │ │课程  │ │成绩  │ │考勤  │ │财务  │  │
│  │服务  │ │服务  │ │服务  │ │服务  │ │服务  │  │
│  └──────┘ └──────┘ └──────┘ └──────┘ └─────┘  │
└───────────────────────┬─────────────────────────┘
                        │
┌───────────────────────▼─────────────────────────┐
│                 数据层                            │
│     MySQL (主库)  │  Redis (缓存)  │  ES (搜索)   │
└─────────────────────────────────────────────────┘
```

### 5.2 推荐技术栈

| 层次 | 技术选型 |
|------|----------|
| **前端** | Vue 3 / React + Element Plus / Ant Design + Axios |
| **后端** | Spring Boot / Django / FastAPI |
| **数据库** | MySQL 8.0 + Redis 缓存 |
| **搜索** | Elasticsearch（可选） |
| **文件存储** | 阿里云 OSS / MinIO |
| **消息队列** | RabbitMQ / RocketMQ（可选） |
| **网关** | Spring Cloud Gateway / Nginx |
| **部署** | Docker + Docker Compose / K8s |
| **CI/CD** | Jenkins / GitHub Actions |

## 7. 典型技术难点与解决方案

| 难点 | 解决方案 |
|------|----------|
| 排课冲突检测 | 使用区间树/位图算法检测时间冲突 |
| 高并发选课 | Redis 预扣库存 + 异步落库 + 限流 |
| 成绩权限控制 | RBAC + 数据权限隔离（按院系/班级） |
| 大量数据查询 | 分库分表 + 读写分离 + 索引优化 |
| 选课时间窗口 | 分布式定时任务（Quartz / XXL-Job） |
| 消息推送 | WebSocket 实时推送 + 消息队列削峰 |

## 9. 总结

学生管理系统是一个成熟且需求明确的教育信息化产品，核心围绕 **学生、教师、课程、成绩、考勤、财务** 六大主线展开。系统需要支持 **多角色权限控制**、**高并发选课场景**、**灵活的成绩计算规则**，并兼顾 **移动端** 和 **数据可视化** 能力。

建议采用 **前后端分离 + 微服务架构**，优先实现核心业务模块（学生管理、课程管理、选课、成绩），再逐步扩展考勤、财务、通知等周边功能。
